# 影片图片模糊设计

## 目标

为影片缩略封面、横版封面和影片剧照提供统一的隐私模糊能力，并在设置页提供可持久化的全局开关。

## 已确认需求

- 模糊开关首次安装时默认开启。
- 模糊范围仅包含影片 `thumb`、影片 `cover` 和影片剧照。
- 演员头像不受该开关影响。
- 网络图片加载失败后展示的本地占位图保持清晰。
- 开关关闭后，当前已显示的相关影片图片立即恢复清晰。
- 开关状态持久化，应用重启后继续使用用户上次选择。
- 使用固定模糊强度 `sigmaX = 12`、`sigmaY = 12`，不增加强度调节功能。

## 架构设计

### 设置状态

在 `StorageKeys` 中增加影片图片模糊键。`SettingsProvider` 负责读取、修改和持久化布尔状态：

- SharedPreferences 中不存在该键时返回 `true`。
- setter 先更新内存状态并持久化，再通知监听者。
- UI 只通过 `SettingsProvider` 访问该状态，不直接操作 SharedPreferences。

### 图片渲染

扩展 `CachedImage`，增加由调用方明确传入的模糊参数。模糊仅应用于成功加载的网络图片内容：

- 成功加载：使用 `ImageFiltered` 对图片应用高斯模糊。
- 加载中：继续展示现有进度指示器，不模糊。
- 加载失败：继续展示现有图标或本地占位图，不模糊。

`MovieCoverImage` 从 `SettingsProvider` 读取开关，并把状态传给 `CachedImage`。影片剧照使用一个专用的共享图片组件，采用同一数据流。这样卡片、列表、首页和详情页的封面场景统一生效，同时演员头像继续直接使用不带模糊参数的 `CachedImage`。

## 设置页交互

在实际路由使用的 `ProfileSettingsPage` 中增加“影片图片模糊”开关：

- 使用 Material 3 `SwitchListTile`。
- 开关值来自 `SettingsProvider`。
- 用户切换时调用 provider 的异步 setter。
- provider 通知后，设置页及当前组件树中的影片图片同步重建。

## 异常与边界处理

- SharedPreferences 未保存值时始终按默认开启处理。
- 写入失败时沿用 SharedPreferences 返回的异步错误，不静默伪造已保存结果。
- 空地址和网络失败继续使用现有错误占位逻辑，且占位内容保持清晰。
- 本功能不修改 URL 选择、图片缓存、解密、裁剪比例或占位图映射规则。
- 本功能不增加单图临时解锁、点击查看原图或模糊强度调节。

## 测试与验收

### 自动化测试

- `SettingsProvider`：验证默认开启、恢复已保存值、切换后持久化和通知。
- `CachedImage`：验证成功图片启用模糊时使用模糊层，关闭时不使用；错误占位不处于模糊层内。
- `MovieCoverImage`：验证 thumbnail 和 cover 都响应全局开关，同时保留各自占位图。
- 影片剧照组件：验证响应全局开关。
- `ProfileSettingsPage`：验证开关文案、初始值以及点击后的状态和持久化结果。
- 运行相关测试、完整 `flutter test` 和 `dart analyze`。

### 设备验收

通过 ADB 检查以下状态：

1. 默认开启时，影片卡片 thumb、详情 cover 和详情剧照均模糊。
2. 设置页关闭后，上述三类图片均恢复清晰。
3. 重启应用后保持关闭状态。
4. 再次开启后恢复模糊。
5. 演员头像及图片失败占位图始终清晰。

## 成功标准

- 所有影片 thumb、cover 和剧照入口统一遵守一个持久化开关。
- 默认状态为开启。
- 非影片图片与本地占位图不被误模糊。
- 不引入新依赖，不改变现有图片地址选择和占位规则。
- 自动化测试、静态分析和 ADB 验收全部通过。
