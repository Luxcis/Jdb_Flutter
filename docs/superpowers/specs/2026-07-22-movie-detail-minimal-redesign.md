# 影片详情最小重构设计

## 目标

修复影片详情页与参考图不一致、底部常驻抽屉遮挡正文的问题。在不改变现有接口和跨功能架构的前提下，将页面恢复为可完整纵向浏览的详情布局。

## 范围

- 保留 `MovieDetailService` 及现有详情、磁链、短评和推荐请求。
- 仅调整影片详情页面及其回归测试，不修改共享影片卡片、演员卡片或主题。
- 移除常驻 `DraggableScrollableSheet`；磁链、短评和相关清单不再覆盖详情正文。
- 不新增依赖、不新增接口、不实现参考图中当前模型没有的数据或交互。

## 页面结构

页面使用一个 `CustomScrollView`，按以下顺序渲染：

1. 标题栏：标题单行省略。
2. 封面：占满可用宽度，保持内容可辨识。
3. 详情卡：番号、发行日期、时长、导演、片商、系列、评分、操作按钮和人数统计。
4. 类别标签。
5. 演员横向列表。
6. 预告片/剧照横向列表。
7. `TA还出演过` 与 `你可能也喜欢` 横向影片列表；在现有接口只提供一组推荐数据时继续复用该数据，避免扩大接口范围。

所有间距、圆角、颜色和文字样式使用当前 Material 3 主题，并保证窄屏按钮和标签可换行、不产生横向溢出。

## 状态与错误处理

- 主详情加载中、失败和重试逻辑保持不变。
- 附属接口失败仍返回空列表，不影响主详情渲染。
- 空演员、空标签、空剧照或空推荐列表时隐藏对应区块。

## 测试与验收

- 先新增 Widget 回归测试，验证详情信息、类别、演员、剧照、推荐区块顺序，以及页面不存在常驻 `DraggableScrollableSheet`。
- 验证窄屏渲染无 Flutter overflow 异常。
- 运行目标 Widget 测试和静态分析。
- 启动 Android 应用后使用 `adb_tool` 打开影片详情、滚动并截图，确认正文无遮挡且区块顺序与参考图一致。
