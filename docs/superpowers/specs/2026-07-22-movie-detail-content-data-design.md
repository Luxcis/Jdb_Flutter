# 影片详情内容数据与紧凑间距设计

## 目标

在保留现有影片详情 Tabs 结构的前提下，进一步收紧信息卡操作区和类别标签，并修复剧照、演员关联影片、相关推荐、磁链及相关清单的数据来源、解析和状态展示。

## 已确认需求

- 缩小“想看”“看过”“存入清单”与上下元素之间的间距。
- 缩小类别标签的内部留白，不改变其他页面 `TagChip` 的默认样式。
- 基本信息卡片的评分保持影片详情接口返回的数值精度，例如 `"score": "4.33"` 显示为 `4.33`，不强制截成一位小数。
- 修复“预告片 / 剧照”“TA还出演过”“你可能也喜欢”“磁链下载”“相关清单”的接口调用和展示。
- 主详情成功后，磁链与相关清单独立加载；失败时在对应 Tab 展示失败信息和重试入口，不伪装成空数据。
- 剧照、TA出演、相关推荐只读取影片详情接口字段，禁止调用独立接口或使用通用推荐接口回退。
- 保持最小重构，不改动现有四个 Tab 的结构和顺序。

## 已定位根因

1. `/api/v1/movies/{movie_id}/magnets` 实际返回数字类型 `size` 和布尔类型 `hd`。当前 `Magnet.size` 按字符串直接转换，任意一条数据转换异常都会让整个磁链请求被捕获为空列表。
2. V4 详情 `/api/v4/movies/{movie_id}` 已在 `movie` 中提供 `preview_images`、`actor_movies`、`relative_movies`。当前模型只解析剧照，且“TA还出演过”和“你可能也喜欢”错误共用需要登录的 `/api/v1/movies/may_also_like`。
3. “相关清单”仍是静态占位，未调用文档规定的 `/api/v1/lists/related?movie_id={id}`。
4. 附属请求异常目前统一转换为空列表，页面无法区分“确实无数据”和“加载失败”。

## 方案选择

采用“V4 详情为主、专用接口补充”的方案：

- 剧照、TA出演影片、相关推荐读取 V4 详情中的对应字段。
- 上述三类内容与主详情共用同一次请求及同一失败状态，不创建独立加载、错误或重试状态。
- 磁链继续调用专用磁链接口，以保留独立加载和后续分页能力。
- 相关清单调用专用相关清单接口并携带 `movie_id`。
- 不再为详情页调用需登录的通用“猜你喜欢”接口。

该方案与当前 OpenAPI 契约一致，请求数量最少，同时仍允许磁链和相关清单独立失败及重试。

## 布局规格

### 操作按钮区

- 保持按钮最小高度 32、水平内边距 12、`VisualDensity.compact` 和现有横向间距。
- 信息卡主 Column 的常规间距由 8 缩小为 6。
- 移除操作按钮前额外的 4 像素 `SizedBox`。
- 操作按钮后的分隔线固定使用 12 像素高度，减少按钮与统计文字之间的空白。
- 仍允许三个按钮在窄屏自动换行。

### 类别标签

- `TagChip.compact` 增加详情页专用的零外层 padding 和水平 6 像素 `labelPadding`。
- 保留 `VisualDensity.compact`、`MaterialTapTargetSize.shrinkWrap` 和 `labelSmall`。
- 默认 `TagChip` 不设置上述紧凑内边距，避免影响其他页面。

### 评分

- 继续使用现有可空 `double` 模型，兼容接口返回的字符串或数字评分。
- 信息卡不使用 `toStringAsFixed(1)`；按解析后的原始有效精度展示，因此 `4.33` 不再显示为 `4.3`。
- 本次只调整影片详情信息卡，不改变其他影片卡片或排行榜的评分格式。

## 数据模型与解析

### MovieDetail

在 `MovieDetail` 中增加：

- `actorMovies: List<MovieSummary>`，来源 `actor_movies`。
- `relativeMovies: List<MovieSummary>`，来源 `relative_movies`。

`screenshots` 继续作为剧照 URL 列表，但规范化层需要兼容：

- 字符串 URL。
- `{url: ...}` 或 `{image_url: ...}` 对象。
- 按分组 Map 包裹的图片数组。

所有关联影片统一通过 `normalizeMovieSummaryJson` 解析，确保 `thumb_url`、`cover_url` 和数字字段兼容现有影片卡片。

### Magnet

保持现有模型字段范围，规范化时：

- `name` 映射为标题。
- `size` 为数字时按 MB 解释并转换为可读单位：小于 1024 显示 MB，大于等于 1024 换算为 GB；字符串大小保持原值。
- `created_at` 映射为发布日期。
- `hd` 严格转换为高清布尔值，不再以“字段存在”判断为 true。

### ListModel

复用现有 `ListModel`，增加规范化映射：

- `movies_count` → `movie_count`。
- `views_count` → `viewed_count`。
- ID 和名称安全转换为字符串。

## 加载与状态管理

主详情请求仍决定页面主体是否可显示。详情成功后：

- 剧照、TA出演影片、相关推荐直接从 `MovieDetail` 渲染，不发额外请求。
- 磁链和相关清单分别发起独立请求。
- 短评保持现有独立请求，但不在本次改变其展示样式。

磁链和相关清单分别维护 `loading / data / error` 状态：

- 加载中：Tab 内容居中显示进度指示器。
- 成功且为空：显示“暂无磁链”或“暂无相关清单”。
- 成功且有数据：显示对应列表。
- 失败：显示 `ErrorRetryWidget`，重试只刷新当前数据源，不重新加载主详情。

主详情首次展示不等待附属请求完成。页面销毁后不再写入状态。

## 展示设计

### 预告片 / 剧照

沿用横向剧照列表和现有模糊设置。只要 `preview_images` 可解析出 URL 就展示；空数据时隐藏整个区块。

### TA还出演过与推荐

- “TA还出演过”只使用 `actorMovies`。
- “你可能也喜欢”只使用 `relativeMovies`。
- 两者继续复用横向 `MovieCard` 列表，空数据时分别隐藏，不互相复用数据。

### 磁链下载

每条磁链显示标题，副标题显示可用的大小与日期；高清项增加简短“高清”标识。缺失字段不渲染空占位文本。

### 相关清单

使用纵向列表展示片单名称，以及影片数和浏览数。当前路由没有片单详情页，因此本次只展示，不增加无效点击行为。

## 组件边界

- `api_data.dart`：只负责不可信 API 数据的安全规范化。
- `MovieDetailService`：负责磁链和相关清单请求，不持有 UI 状态。
- `_MovieDetailPageState`：分别维护磁链、短评、相关清单的异步状态和重试方法。
- `_BasicInfoTab`：使用详情模型中的三类内容，不再接收通用推荐请求结果。
- `_MagnetList` 与新增相关清单列表：只根据显式状态渲染，不自行请求数据。

不引入新依赖，不拆分现有详情 screen 文件，不扩展到操作按钮业务逻辑或片单详情路由。

## 测试策略

### 数据与服务测试

- V4 详情能解析字符串、对象和分组 Map 形式的剧照。
- `actor_movies` 与 `relative_movies` 分别进入正确字段。
- 数字磁链大小和布尔 `hd` 不再导致整批解析失败。
- 相关清单请求携带 `movie_id`，并兼容 `movies_count/views_count`。

### Widget 测试

- 操作按钮与上下内容的间距符合新规格。
- 紧凑类别标签使用明确的内部 padding，默认标签不受影响。
- 接口评分 `"4.33"` 在基本信息卡中显示为 `4.33`。
- 两个推荐区块展示不同的数据。
- 页面不请求 `/api/v1/movies/may_also_like`，也不为剧照、TA出演或相关推荐发起其他请求。
- 磁链展示标题、大小、日期和高清标识。
- 相关清单展示名称与统计。
- 磁链或相关清单失败时显示独立重试；点击重试只请求对应接口。
- 空数据与失败状态文案不同。

### 完整验证

- 运行相关测试和完整 `flutter test`。
- 运行 `dart analyze`。
- 安装到已连接设备，通过 ADB 检查间距、剧照、两类推荐、磁链、相关清单、重试状态和 Flutter 异常日志。

## 成功标准

- 操作按钮上下留白和类别标签内部留白明显收紧，窄屏无溢出。
- V4 详情中的剧照、演员关联影片和相关推荐各自正确展示。
- 实际返回数字大小的磁链能正常解析并展示。
- 相关清单按影片 ID 请求并展示真实数据。
- 附属接口失败可识别、可独立重试，不影响主详情。
- 自动化测试、静态分析和 ADB 验收全部通过。
